# 知识库分类地图

回答前先按语义判断问题应检索哪些知识库。不要只按关键词机械匹配；关键词只是线索，真正依据是用户想解决的问题类型。

## 主分类

### 01_考据

用途：回答古人怎么做、文献怎么说、概念有没有出处、某种说法是否有古籍依据。

典型材料：

- 古籍原文
- 校勘
- 训诂
- 礼制
- 历史文献
- 图像材料
- 制度史

优先触发问题：

- “古人是不是这样射？”
- “这个说法有没有出处？”
- “礼射文献怎么说？”
- “某个术语是什么意思？”

### 02_数据

用途：回答现代实测、性能、数值、曲线、拉感、箭重、效率。

典型材料：

- 实测数值
- 形变曲线
- 拉感评分
- 箭重
- FOC
- 效率
- 器材性能记录

优先触发问题：

- “实际效果怎么样？”
- “这个弓效率如何？”
- “箭重应该多少？”
- “有没有实测数据？”

### 03_文化

用途：回答理念、意义、对外表达、社群精神、礼射精神。

典型材料：

- 礼射
- 不射之射
- 内志外体
- 修身观念
- 社群精神表达
- 对外文案

优先触发问题：

- “为什么要参加？”
- “古典射学的意义是什么？”
- “怎么对外解释？”
- “怎么回应质疑但不吵架？”

### 04_新手

用途：回答新手入门、安全、第一步怎么做、常见误区。

典型材料：

- 安全规范
- 入门指南
- 动作基础
- 误区纠偏
- 第一次参加活动说明

优先触发问题：

- “新手怎么开始？”
- “第一次参加要注意什么？”
- “我适合参加吗？”
- “第一步练什么？”

### 05_社群

用途：回答章程、活动规则、报名、费用、场地、安全、记录会、联盟治理。

典型材料：

- 章程
- 活动手册
- 场地布置
- 群内共识
- 对外说明
- 公示文件
- 记录会规则

优先触发问题：

- “现在规则是什么？”
- “活动能不能这样办？”
- “费用能不能盈利？”
- “怎么报名/承办？”
- “联盟允许吗？”

### 06_器材

用途：回答弓、箭、护具、材料、形制、工艺、选弓。

典型材料：

- 弓梢形制
- 角弓
- 竹木弓
- 明弜
- 登龙
- 漆艺
- 囊韬
- 箭矢与护具
- 制作工艺

优先触发问题：

- “怎么选弓？”
- “这个弓形合不合适？”
- “明弜/登龙是什么？”
- “材料和性能有什么区别？”

## 辅助分类

### 07_问答解释

用途：提供自然回答骨架，让回答不像直接摘素材。

注意：

- 不能单独作为最终事实依据。
- 必须用事实层 KB 校准。
- 当解释层和事实层冲突时，以事实层为准。

### 99_待审核

用途：暂存来源、隐私、可信度或分类尚未确认的材料。

注意：

- 默认不用于公开回答。
- 只能作为待核线索。
- 不能写成定论。

## 语义路由规则

- 器材形制、材料、性能、选弓、弓梢、明弜、登龙、漆艺：查 `01_考据 + 02_数据 + 06_器材`，必要时补 `05_社群` 看活动规则。
- 古人怎么做、文献怎么说、某概念是否有出处：优先查 `01_考据`。
- 当下怎么选、怎么配、实际效果如何：优先查 `02_数据 + 06_器材`。
- 社群规则、比赛、活动安排、场地、安全、报名、费用、承办：优先查 `05_社群`。
- 新人如何开始、是否适合、第一步练什么：优先查 `04_新手`，必要时补 `05_社群` 的安全规则。
- 理念、意义、礼射精神、对外表达：优先查 `03_文化`，必要时补 `01_考据`。
- 用户语气激烈、质疑规则正当性、带引战倾向：优先查 `05_社群 + 03_文化`，必要时补 `01_考据` 或 `02_数据`。

## 整合规则

- 不同库给出的材料要分清适用场景。
- 问历史事实，以 `01_考据` 为主。
- 问当下使用，以 `02_数据 + 06_器材 + 05_社群安全规则` 为主。
- 问联盟活动是否允许，以 `05_社群` 为准。
- 问意义和对外表达，以 `03_文化` 为主，再补考据。
- 检索不到对应库材料时，要明确说“当前知识库没有足够材料支持”，不能用别的库硬凑答案。
