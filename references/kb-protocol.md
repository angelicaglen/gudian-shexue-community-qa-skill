# 知识库使用协议

回答前先判断问题类型，并按类型检索知识库。

## 六大主分类

- 考据：古籍原文、校勘、训诂、礼制、历史文献、图像材料。
- 数据：实测数值、形变曲线、拉感评分、箭重、效率、器材性能记录。
- 文化：礼射、不射之射、内志外体、修身观念、社群精神表达。
- 新手：安全规范、入门指南、动作基础、误区纠偏。
- 社群：章程、活动手册、场地布置、群内共识、对外说明。
- 器材：弓梢形制、角弓、竹木弓、明弜、登龙、漆艺、囊韬、箭矢与护具。

## 路由规则

- 问器材形制、材料、性能、选弓、弓梢、明弜、登龙、漆艺时，触发三联查：考据 + 数据 + 器材。
- 问古人怎么做、文献怎么说、某概念是否有出处时，优先查考据。
- 问当下怎么选、怎么配、实际效果如何时，优先查数据和器材。
- 问社群规则、比赛、活动安排、场地、安全、报名时，优先查社群。
- 问新人如何开始、是否适合、第一步练什么时，优先查新手。
- 问理念、意义、礼射精神、对外表达时，优先查文化，并视情况补充考据。
- 用户语气激烈、质疑规则正当性、带有引战倾向时，优先查社群与文化，必要时再补充考据或数据。

## 检索策略

对于单一分类问题，先用最相关分类检索。如果结果不足，再扩大到相关分类。

对于复杂问题，分成多个查询。例如“某种弓是否适合联盟活动”应拆为：

1. 器材形制和性能。
2. 社群章程和活动规则。
3. 安全或新手使用边界。

若 API 只支持单分类检索，多次调用 `/search`，不要用一次宽泛查询替代所有证据。
