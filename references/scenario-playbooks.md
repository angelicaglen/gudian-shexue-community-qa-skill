# 古典射学 reasoning 场景策略

> 状态：v1，作为问答 skill 的场景判断与争议回应参考。
> 原则：场景策略像判例，随知识库和案例扩充；不得覆盖第一性原理和正式知识库证据。

## 通用格式

每个场景包含：
- 触发信号
- 必查知识库
- 判断边界
- 禁忌
- 推荐回答方向

## 场景策略使用顺序

场景策略只负责“像判例一样提醒怎么处理相似问题”，不能替代第一性原理和知识库证据。

使用顺序：

```text
先识别场景
再查相关知识库
再用第一性原理校准判断
再选择普通问答、社群回复、对外文案或审计模式
```

同一个争议场景有两种完全不同的输出：

- 用户自己想理解：解释规则、原因和依据，不给“建议回复”。
- 用户要发给别人：生成可复制回复，先降温，再讲事实和边界。

不确定时，先问用户要哪一种，不要自动代写。

## 1. 撒放方式争议

触发信号：
- 蒙古式、拇指撒放、三指撒放、地中海式、扳指、指机、正统、不准参加。

必查知识库：
- 技术路线与规则补充。
- 活动章程、检录规则、器材/指机规范。
- 相关考据材料，如宋明射学经典整理材料。

判断边界：
- 联盟正式口径不应使用“蒙古式拇指撒放”作为规则术语。
- 联盟活动规范要求拇指撒放，并使用规范扳指或指机。
- 地中海式三指撒放不属于联盟相关活动中的合格技术形式。
- 该规则用于活动准入、技术规范和检录边界，不用于流派羞辱。

禁忌：
- 不说“蒙古式才正统”。
- 不把其他射箭体系贬为“假传统”。
- 不把未入库的群聊争论当成规则来源。

## 2. 器材原教旨争议

触发信号：
- 天然竹木角弓、层压弓、碳箭、假传统、不配参加、器材鄙视链。

必查知识库：
- 活动器材规范。
- 弓形、弓把、箭重、FOC、箭长、箭羽、箭头等规则。
- 器材考据和实测资料。

判断边界：
- 能否参加活动看具体活动章程和检录规范。
- 器材传统性可以讨论，但不能用标签攻击参与资格。
- 若某材料或形制被章程明确允许或禁止，按章程说。

禁忌：
- 不用“真传统/假传统”替代规则。
- 不用单一材料标准压倒活动安全和检录规则。

## 3. 活动收费争议

触发信号：
- 公益性、报名费、黑心钱、盈利、成本、场地费、饮用水、靶纸。

必查知识库：
- 联盟公益性、非竞技性原则。
- 活动筹办文件中的费用说明。

判断边界：
- 公益性不等于零收费。
- 合理覆盖运营成本和劳动支出可以讨论。
- 不能以超额盈利、奖金、奖品、竞技淘汰为核心。

禁忌：
- 不把合理成本说成牟利。
- 不替未公开账目背书。

## 4. 跨组织对线争议

触发信号：
- 外部组织不承认、规则瞎编、智商税、公开拉踩、认证争议。

必查知识库：
- 联盟章程、活动规则、记录方式、对外说明。

判断边界：
- 联盟规则首先服务自身活动治理、安全和记录标准。
- 不强迫其他组织承认，也不贬低其他体系。
- 对外回应应回到公开规则和事实，不与对方互骂。

禁忌：
- 不代表联盟发表未授权官方声明。
- 不用攻击性语言回击。

## 5. 新老参与门槛争议

触发信号：
- 新手、练满一年、40 磅、没资格、凑热闹、捣乱、报名门槛。

必查知识库：
- 活动报名规则。
- 安全规范、现场管理、违规处理。
- 新手入门材料。

判断边界：
- 不能把经验建议说成联盟通用硬门槛。
- 某活动可以有具体门槛，但必须以活动章程或后续公示为准。
- 安全能力、规则理解和现场服从比资历标签更重要。

禁忌：
- 不用资历羞辱新人。
- 不替具体活动编造报名门槛。

## 6. 普通问答 vs 社群回复

触发信号：
- 用户描述“群里吵/有人说/争议”，但没有明确要求代写。

判断边界：
- 争议背景不等于社群回复需求。
- 默认用户是在自己理解规则。
- 若不确定，先问：你是想自己弄明白规则，还是要我写一段可直接发到群里的回复？

禁忌：
- 不默认输出“可以这样回/建议回复/群里可以说”。
- 不在普通问答中自动生成公告、声明或评论区文案。

推荐回答方向：
- 普通问答：直接解释“这件事规则上怎么判断、为什么、依据是什么”。
- 社群回复：用户确认后再输出可复制文本；语气要降温、去攻击、去身份羞辱。
- 对外发布：只使用公开或脱敏材料，避免内部群聊细节和个人信息。

## 8. reasoning 扩充请求

触发信号：
- 用户说“这个案例以后也要会判断”“把这个加到 skill 里”“第一性原理要不要扩充”“分析工具要随知识库扩充”。

判断边界：
- 问答 skill 不能直接把临时对话写成正式 reasoning 真源。
- 如果只是当前对话里的想法，应标注为待核。
- 如果来自新增知识库材料、正式规则补充或多次稳定案例，应建议走清洗入库 skill 生成 proposal。

推荐回答方向：
- 先区分：这是临时表达优化、场景策略补充、分析工具补充，还是第一性原理修订。
- 第一性原理修订最重，必须人工明确批准。
- 场景策略和分析工具可以更频繁更新，但仍要经过更新部署 skill 发布。

禁忌：
- 不让问答 skill 自我修改 reasoning。
- 不把用户临时口述当成已经入库的权威规则。
- 不把哲学分析变成事实来源。

## 7. 深度解释模式

触发信号：
- 为什么这个圈子会这样、背后结构是什么、从社群治理角度看、从文化/哲学层面分析、这个争议为什么总吵起来。

判断边界：
- 可以启用第一性原理和分析工具。
- 不默认点名哲学理论，除非用户要求。
- 最后要落回可执行建议或清晰判断。

推荐结构：

```text
表层问题：……
深层结构：……
关键误区：……
更稳的判断：……
可以怎么做：……
```
