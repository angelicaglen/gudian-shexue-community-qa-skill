# 古典射学 reasoning 分析工具

> 状态：v1，供问答 skill 内部分析使用。
> 原则：哲学与社会理论只作为内部工具，不默认在回答中显名，不把用户问题写成论文。

## 使用方式

- 普通规则问题优先查知识库，不默认启用深度分析。
- 当问题涉及定义权、传统边界、社群争议、讨论质量、文化解释、技术路线时，可启用本框架。
- 分析工具不能替代证据。没有知识库来源时，不得把分析写成事实定论。
- 分析工具必须服务第一性原理：先判断要保护的实践传统、规则边界和讨论秩序，再选择工具解释。
- 不要把所有深度问题都写成哲学论文。默认只输出分析后的清晰结论，除非用户明确要求理论展开。

## 可扩展原则

分析工具不是固定清单。随着知识库扩充，可以增加新的工具或细化旧工具，但必须经过清洗入库与更新部署流程。

新增或修订分析工具的触发条件：

- 新材料反复出现某类稳定问题，例如术语误用、规则冲突、社群治理、技术复原、文化解释。
- 现有工具无法解释用户常问问题，导致回答只能堆材料或给空泛结论。
- 新增正式规则、活动复盘或争议案例，需要沉淀成可复用判断方法。

不能新增工具的情况：

- 只是为了让回答显得更“高级”。
- 只是单次聊天里的灵感，没有知识库材料支撑。
- 会覆盖正式章程、规则、公示文件或实测数据。

新增工具应写清：

```text
工具名称
解决什么问题
适用信号
内部判断问题
不能越过的证据边界
普通问答时如何说成人话
```

## 工具一：语言游戏

用于判断术语属于哪个话语体系，以及规则如何在共同体中被使用。

适用问题：
- “蒙古式拇指撒放”是否是联盟正式术语？
- “正统”“传统”“古典形制”在联盟语境中是什么意思？
- 外部射箭圈术语能不能直接替代联盟规则术语？

内部问题：
- 这个词属于哪个话语体系？
- 联盟规则是否使用这个词？
- 联盟内部更准确的规范表达是什么？

## 工具二：实践传统

用于判断某条规则是否保护古典射学的内在标准，而不是只服务外部结果。

适用问题：
- 为什么古典射学不是现代竞技射箭的复古包装？
- 为什么活动规则要限制器物、撒放、扳指或指机？
- 为什么“射中”不是唯一评价维度？

内部问题：
- 这个问题是在问外部结果，还是实践内部标准？
- 规则是在保护传统，还是制造身份排斥？

## 工具三：默会知识

用于处理那些不能完全靠文字规则讲清楚、但又必须转化为训练与检录边界的技术。

适用问题：
- 撒放、拉距、站姿、指机使用为什么需要示范与训练？
- 什么可以写成规则，什么只能作为训练路径？

内部问题：
- 这是能写成规则的知识，还是需要示范和训练的默会知识？
- 最低可公开标准是什么？

## 工具四：交往理性

用于处理社群争议、群聊冲突和公共讨论质量问题。

适用问题：
- 为什么群里讨论会退化成身份攻击？
- 如何让争议回到证据、规则和共同事实？
- 何时需要先确认用户是自己理解，还是要写社群回复？

内部问题：
- 这个争议是在求事实，还是在争身份/道德高地/话语权？
- 讨论有没有共同承认的证据标准和发言规则？

## 工具五：话语权与可复核定义

用于处理联盟定义权、规则合法性和外部质疑。

适用问题：
- 联盟凭什么定义古典射学？
- 定义权会不会变成个人权威？
- 如何回应“规则是自己瞎编的”？

内部问题：
- 这个规则划定了什么边界？
- 它保护什么，也排除了什么？
- 排除是否有公开依据、可复核来源和安全/文化理由？

## 工具六：技术-经验校准

用于处理文献、现代复原、实测数据、安全规则和活动执行之间的关系。

适用问题：
- 文献形制、现代复原和实测性能冲突时怎么说？
- 规则参数是否能压倒安全要求？
- 器材争议中，哪些是规则，哪些是经验，哪些是数据不足？

内部问题：
- 这个判断来自文献、数据、规则，还是现场经验？
- 是否把单一证据误当成所有场景的唯一答案？

## 输出原则

默认输出不显名哲学理论。可使用以下结构说人话：

```text
表面看，这是……
更深一层，它其实是……
所以处理时不能只靠……
更稳的边界是……
```

只有用户明确要求哲学分析时，才可以点名具体理论。
