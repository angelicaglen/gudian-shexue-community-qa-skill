# 古典射学社群问答 skill 调用测试报告

测试时间：2026-07-14

## Goal

重新测试古典射学社群问答 skill 的知识库调用链路，覆盖 Cloudflare 主 API 与 CloudBase 国内备用 API，验证检索、路由、引用和回答边界。

## Workflow

1. 验证 Cloudflare 主 API 与 CloudBase 备用 API 的 `/health` 和 `/route-search`。
2. 用 10 个从贴近到泛的问题运行 `scripts/smoke-test.mjs`。
3. 修复调用层：增加 CloudBase 自动回退、请求超时、smoke test 脚本。
4. 复测全局 skill 目录与项目内开源 skill 目录。

## 修复结果

- `query-kb.mjs` 默认按 Cloudflare -> CloudBase 顺序调用。
- 主 API 超时或失败时，会自动尝试 CloudBase 备用 API。
- 新增 `--timeout-ms` 与 `GUDIAN_SHEXUE_KB_TIMEOUT_MS`。
- 新增 `GUDIAN_SHEXUE_KB_API_URLS`，可配置多个候选 API。
- 新增 `scripts/smoke-test.mjs`。
- skill 版本更新为 `0.1.3`。

## 审计结论

调用链路已经可用，CloudBase 备用 API 已接入 skill 调用层。

主要问题不在 API 或 skill，而在知识库内容层：`文化`、`考据`、`新手`、`数据`、`器材` 分类材料不足。现在很多泛问题仍依赖 `07_问答解释` 生成自然回答骨架，但事实层证据不够完整。

后续优先补库顺序：

1. `04_新手`：第一次参加、活动安全、入门说明。
2. `06_器材 + 02_数据`：选弓、箭重、FOC、箭长、器材规范。
3. `01_考据 + 03_文化`：礼射、古典射学意义、对外解释。
