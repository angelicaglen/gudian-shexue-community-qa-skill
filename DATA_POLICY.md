# 数据收集与使用政策

本政策说明古典射学社群问答 Skill 免费测试版在调用公共知识库 API 时会收集哪些数据、为什么收集、如何使用，以及用户需要注意什么。

继续使用免费测试版公共 API，即表示你理解并同意本政策。

## 1. 收集哪些数据

免费测试版可能收集：

- 用户问题文本；
- 模型最终回答；
- 回答使用的知识库来源、链接、作者、发布时间、KB 编号、段落号和时间点；
- 用户反馈、错误反馈和质量标记；
- interaction_id、session_id、skill 版本、客户端自报模型信息；
- 调用时间、API 版本、服务错误、超时和必要的诊断信息。

## 2. 不主动收集哪些数据

本项目不主动采集：

- 手机号；
- 微信号、QQ、邮箱等通讯方式；
- 真实姓名；
- 身份证号、证件号；
- 住址、精确位置；
- 支付信息；
- 与古典射学学习无关的个人身份资料。

请不要在提问中输入个人隐私或他人隐私。若用户自行输入隐私内容，可能随问题文本一并提交；项目会尽量进行脱敏、清理和最小化保存，但无法保证在提交前自动识别所有隐私内容。

## 3. 数据用途

收集的数据仅用于：

- 改进古典射学知识库；
- 发现缺失材料、错别字、转写错误和术语误识别；
- 优化检索、路由、引用和回答生成质量；
- 审计高风险回答；
- 排查 API 故障、异常调用和滥用行为；
- 统计公共服务的基本运行质量。

本项目不会出售原始用户问答数据，不会将原始问答数据用于第三方广告投放或身份追踪。

## 4. 数据处理原则

项目会尽量遵循以下原则：

- 最小必要：只保留改进知识库和服务质量所需的数据。
- 去身份化：不主动记录可识别真实身份的信息。
- 脱敏优先：发现隐私、联系方式、地址等内容时尽量删除或遮蔽。
- 来源可查：用于知识库改进的内容应尽量保留公开来源、时间点和审核状态。
- 审核入库：用户问答日志不会自动成为正式知识库事实，进入知识库前需要整理、来源标注和审核。

## 5. 用户提交资料

如果你主动提交史料、视频链接、转写纠错、术语修正、活动资料或其他知识库共建内容，即表示你确认：

- 你有权提交该内容，或该内容来自公开可引用来源；
- 你允许项目方对内容进行整理、脱敏、改写、标注来源和纳入知识库候选；
- 项目方可因来源不清、版权风险、隐私风险、质量不足或不适合项目方向而拒收或延后收录。

投稿进入正式知识库前，项目方会尽量保留来源、作者、链接、发布时间和审核状态。

## 6. 用户选择

如果你不同意免费测试版的数据收集，请不要使用默认公共 API。你可以：

- 使用项目源代码自部署私有 API；
- 在私有部署中关闭 interaction log；
- 避免输入任何个人隐私或无关信息；
- 向项目方反馈需要删除或脱敏的具体内容。

## 7. 删除与更正

如果你发现日志、知识库候选或正式知识库中包含隐私、错误来源、侵权内容或明显错误，可以联系项目方请求更正、脱敏或删除。

为了定位问题，请尽量提供：

- 问题发生时间；
- 相关问题或回答片段；
- 来源链接或 KB 编号；
- 需要更正、脱敏或删除的原因。

## 8. 政策更新

本政策可能随项目发展更新。继续使用免费测试版公共 API 视为接受更新后的政策。重大变更会尽量通过 README、版本文件、项目公告或 skill 提示说明。
